Key Concepts

  • HTTP: The Web’s protocol for request/response between a client and a server.

  • HTTPS: HTTP with encryption (TLS) for additional privacy and integrity.

  • Client & Requests : The client is the requester (usually a browser or an app). The request is a method + target (URL/path) + headers + optional body.

  • Server & Responses: The server is the responder that processes requests and sends back data. The response is a status code + headers + optional body.

  • Methods & Status codes: Methods include GET, POST, PUT, DELETE, PATCH, HEAD. The different status codes include 2xx success, 3xx redirect, 4xx client error, 5xx server error.

  • Versions: HTTP/1.1, HTTP/2, HTTP/3 same model, each is progressively faster.
