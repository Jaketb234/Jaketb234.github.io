Home

HyperText Transfer Protocol (HTTP) is the baseline application-layer protocol for data transfer on the World Wide Web. It is one of the main protocols that help networked devices interact and share data. HTTP is a client-server protocol, which means it works based on requests and responses. Not only can it fetch Hypertext documents, but it is also used to post content to servers or to get an image or video. Over time, HTTP has evolved to become one of the most important protocols for efficient networking.

Key Terms

  • HTTP — A request/response conversation between a client and a server.
  • Client — The requester (usually a browser or app).
  • Server — The responder that hosts content or APIs.
  • Request — Method + target URL + headers + optional body sent by the client.
  • Response — Status code + headers + optional body sent back by the server.
  • Method — The action (GET, POST, PUT, DELETE, PATCH, HEAD).
  • Status code — Code displayed in result of the request (2xx success, 3xx redirect, 4xx client error, 5xx server error).
  • Headers — Metadata like Content-Type, Cache-Control, Authorization, Cookies.
  • Cache — Digital storace center for reuse of a stored copy.
  • Versions — HTTP/1.1 (baseline), HTTP/2 and HTTP/3 (more current).
